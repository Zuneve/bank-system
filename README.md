# Bank service

## Что реализовано

- Многомодульная структура Maven.
- Хранение пользователей, счетов и операций в PostgreSQL.
- Data Access слой на Hibernate / JPA.
- Service слой с бизнес-логикой.
- Консольное приложение для ручной проверки.
- Unit-тесты бизнес-логики с Mockito-моками, без подключения к БД.

## Модули

- `domain` - доменные JPA-сущности:
  - `User`;
  - `Account`;
  - `Operation`;
  - `HairColor`;
  - `OperationType`.
- `repositories` - слой доступа к данным:
  - JPA-репозитории;
  - `persistence.xml`;
  - настройки подключения к PostgreSQL.
- `bank-service` - бизнес-логика:
  - создание пользователей;
  - создание счетов;
  - пополнение;
  - снятие;
  - переводы;
  - расчёт комиссий;
  - управление друзьями;
  - unit-тесты.
- `bank-console` - консольная точка входа.

## Бизнес-логика

Поддерживаются операции:

- создание пользователя;
- поиск пользователя по id;
- поиск пользователя по login;
- добавление друга;
- удаление друга;
- создание счёта;
- просмотр баланса;
- пополнение счёта;
- снятие со счёта;
- перевод между счетами.

Комиссии при переводе:

- между своими счетами - `0%`;
- другу - `3%`;
- остальным пользователям - `10%`.

При переводе получателю зачисляется исходная сумма перевода, а с отправителя списывается сумма перевода плюс комиссия.

## PostgreSQL

PostgreSQL запускается через Docker Compose.

Внешний порт задан как `5433`, потому что локальный PostgreSQL может уже занимать `5432`.

Параметры подключения:

- database: `bank_db`;
- user: `bank_user`;
- password: `bank_password`;
- JDBC URL: `jdbc:postgresql://localhost:5433/bank_db`.

Запуск БД:

```bash
docker compose up -d
```

Если нужно полностью пересоздать БД:

```bash
docker compose down -v
docker compose up -d
```

Проверка подключения:

```bash
docker compose exec postgres psql -U bank_user -d bank_db -c "select current_user, current_database();"
```

## Сборка и тесты

Запуск всех тестов:

```bash
mvn clean test
```

Тесты бизнес-логики находятся в модуле `bank-service`.

Они проверяют:

- успешное снятие денег;
- ошибку при недостаточном балансе;
- пополнение счёта;
- перевод между своими счетами;
- перевод другу;
- перевод другому пользователю;
- сохранение истории операций.

Тесты используют Mockito-моки репозиториев и не зависят от PostgreSQL/Hibernate.

## Запуск консольного приложения

Сначала собрать и установить модули:

```bash
mvn -pl bank-console -am install -DskipTests
```

Затем запустить консоль:

```bash
mvn -f bank-console/pom.xml exec:java
```

В консоли доступны команды:

```text
1. Create user
2. Create account
3. Add friend
4. Delete friend
5. Find user by id
6. Find user by login
7. Get account balance
8. Deposit
9. Withdraw
10. Transfer
0. Exit
```

## Пример ручной проверки

1. Создать пользователя через `Create user`.
2. Скопировать UUID пользователя.
3. Создать счёт через `Create account`.
4. Скопировать UUID счёта.
5. Пополнить счёт через `Deposit`.
6. Проверить баланс через `Get account balance`.
7. Создать второго пользователя и второй счёт.
8. Выполнить `Transfer`.
9. Проверить балансы обоих счетов.

## Важные технические решения

- Старые in-memory реализации не используются в текущей точке входа.
- Консольное приложение создаёт JPA-репозитории и сервисы вручную, без Spring.
- Бизнес-логика расчёта баланса и комиссий находится в service layer.
- DAO слой отвечает за сохранение и поиск сущностей.
- Entity-классы явно перечислены в `persistence.xml`, чтобы Hibernate корректно видел сущности из отдельного модуля.
- Таблица пользователей называется `users`, а не `User`, потому что `User` конфликтует с PostgreSQL.
- История операций хранится в таблице `operations`.
